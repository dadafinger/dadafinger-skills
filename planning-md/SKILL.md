# Planning — MD

Product 기획 문서(spec md)를 **신규 생성**하거나 **기존 수정**한다. 진입 시 **어느 쪽인지 먼저 되묻는다**(명시돼 있으면 생략).

작성 포맷 정본: **`assets/명세서_템플릿.md`**(Screen ID·변경이력·진입경로·영역별 명세 `### N · 기능요소명 <!-- marker=… -->`·QA). 이 섹션 계약을 html·figma가 그대로 소비(4중 정합)하므로 **여기서 섹션 경계를 확정**한다.

## A. 신규 생성
1. **초안 요구** — 어떤 내용을 담을지 사용자에게 **초안을 요청**한다(무슨 화면/기능, 핵심 항목). 초안 없이 임의로 채우지 않는다.
2. **필수정보 확인** — 초안과 함께 묻는다: **파일 제목/파일명**, 버전(`vN`), **저장 위치**(미지정 시 물어봄), 대상 화면/기능명.
3. **생성** — 초안 기반으로 `명세서_템플릿.md` 포맷에 맞춰 작성. "산출물 규약" 준수(상단 changelog 표 + 끝에 Open Issues). 미결·결정 대기 항목은 본문에 섞지 말고 Open Issues로 분리.
4. **보고** — 생성 경로 + 담은 내용 요약 + Open Issues.

## B. 기존 수정 (복제 후 버전업)
⚠️ **원본을 덮어쓰지 않는다.** 기존 파일은 보존하고 **복제본을 버전 +1 한 새 파일**에 반영한다. 그 새 파일이 결과물.

1. **대상 파일 되묻기** — 수정할 md 경로(인자)를 묻는다. 지정돼 있으면 생략.
2. **수정사항 수신** — 자연어·캡처 등 **정해진 양식 없이** 받는다.
3. **대상 md 읽기** — 현재 구조·버전·changelog 파악(편집 전 필수).
4. **항목화 & 비판적 검토** — 각 수정사항을 *어느 섹션*에 반영할지 매핑하고, 충돌·번복(직전 결정 뒤집기)·수치/용어 불일치·실현가능성·해석 모호를 잡는다. **모호하면 「확인 필요」로 모아 한 번에 질문**(추측 적용 금지).
5. **복제 후 버전업** — 원본은 건드리지 않고, 같은 폴더에 **버전 +1 한 새 파일**(예: `… v6.md` → `… v7.md`)을 만들어:
   - 제목·버전 라벨·changelog 최상단 행을 새 버전으로 갱신(이전 이력 보존).
   - 본문에 수정사항 반영, ⭐버전 마커로 변경 지점 표시.
   - 변경되지 않은 외부 산출물(프로토타입 html·Figma 노드 등) 참조는 실제 버전 그대로, 차이 있으면 명시.
6. **보고** — **새 파일 경로** + 수정사항 → 반영 위치 → 조치(반영/보류/질문) 표. 보류·질문은 맨 앞.

**가드** — 원본은 절대 덮어쓰지 않는다(복제본에만 작업). 인자 폴더 밖 다른 파일은 건드리지 않는다. 큰 구조 변경(섹션 삭제·재배치)·충돌 해소는 **새 파일 작성 전 사용자 확인**.

## 산출물 규약
- 버전 `vN`. 문서 **상단 changelog 표**, **끝에 Open Issues 체크리스트**.
- 미정의·결정 대기 항목은 본문에 섞지 말고 **Open Issues로 분리**.
- 화면별 **단독 파일** 우선(각 화면은 화면 ID·Figma 노드와 1:1).
- 기존 문서 수정은 **원본 보존 + 복제본 버전업**. 같은 파일 덮어쓰기 금지.
- **저장 경로(정본)**: `…/00.CONTRASS PLANNING/4.0/<기능>/md/vN/명세서(최신)/<화면>.md`. 즉 버전 폴더(`md/vN`) **바로 아래가 아니라 `명세서(최신)` 하위**에 둔다(같은 레벨에 규칙서는 `md/vN/규칙서/`). 미지정 시 이 규약을 기본값으로 쓰되 사용자에게 확인.

### Product 화면 설계 원칙
1. **설정 vs 발생/조치 화면 분리** — 룰의 *설정 조건*과 *실시간 발생/위반/조치*(알람 센터)를 섞지 않는다. 런타임 표현(현재값·위반 시각·타임라인·조건 트리)은 발생 화면에만, 설정 화면은 정의만.
2. **복합(combination) 엔티티** — 엔티티 단위 단일 유형 분류(THRESHOLD/STATUS 컬럼) 금지. 유형은 **조건 단위 속성**(조건별 배지). 엔티티엔 **구성(`단일`/`복합 N`)** + **충족/위반(`M / N`)**.
3. **발생 시점 스냅샷** — 발생 화면 평가값은 **발생 순간 동결** 저장, 조회/통보 시 재조회 금지. AND "트리거"는 "마지막으로 위반시킨 조건(방아쇠)".
4. **트리거 식별 UX** — ① 위반 강조 ② 현재값·임계값 나란히 ③ 기여도(M/N·방아쇠) ④ 위반 타임라인 ⑤ 조건 트리.
5. **대용량 목록** — (a) 필터 **서버사이드** 전역. (b) 룰 관리(소량)=페이저 / 이벤트(대용량)=keyset 무한 스크롤+시간창+패싯. (c) "전체 선택"=현재 필터 전체 + "N건 적용" 확인.
