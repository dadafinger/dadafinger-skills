# Figma Spec Review — 4대 기준 기계 점검 체크리스트

> 검수 시 **전수 확인**. 각 행 = 점검 1건. `등급 기본값`은 위반 시 부여하는 기본 등급(맥락상 조정 가능). 모든 FAIL/WARN은 **노드 id + 인용 근거** 필수.

## ① 포맷 준수 (Format)

| # | 점검 | 위반 신호 | 등급 기본값 |
|---|---|---|---|
| F1 | 3단 골격 존재 | 헤더행 / 좌 1920 목업 칼럼 / 우 Description 칼럼(2) 중 누락 | FAIL |
| F2 | 헤더행 = `header` 인스턴스(Screen ID·Page Title·Author·Version) | 손그림 텍스트/필드 누락 | FAIL |
| F3 | 검은 풀폭 타이틀 바(Description/Appendix) | 바 없음·색 다름 | WARN |
| F4 | 섹션 = 거터(번호 50px) + title-bar(44px) + 본문 TEXT 노드 | 구조 다름·title-bar 없음 | WARN |
| F5 | 예시 노드 클론 골격(임의 레이아웃 아님) | 손그림 스켈레톤·정렬 깨짐 | FAIL |
| F6 | 본문 = **플랫 인코딩 정본(setFlat)** — 섹션 본문을 1개 TEXT 노드에 `■`/번호/`•`·공백으로 계층 표현 | 네이티브 리스트(listOptions·indentation)로 계층 표현 = 구 A(깨짐) | FAIL |
| F7 | **플랫 마커 규약 준수** — L0 `■ `·L1 `1.`또는`• `·L2 `a.`또는`• `·L3 `• `, 들여쓰기 공백 `0/2/5/8` | 마커·들여쓰기 폭 어긋남·탭/전각 혼용 | WARN |
| F8 | (자체 Changelog 운영 시) Changelog 행 ≥ 2 | 제목만 있고 행 비어 있음 | FAIL |
| F9 | (4.0.0 전역 history 운영 시) 개별 페이지에 Changelog/버전 마커 없음 | `vX.X`·NEW 배지·Intro 버전 라인 잔존 | FAIL |

### 화면 4종 전수 서술 (유형별 필수) — 누락 시 FAIL
> 상세는 figma-desc-spec `references/screen-type-checklists.md`. 핵심만:

- **목록**: LNB/breadcrumb · 헤더 액션 · 속성 필터 · 검색 · 컬럼 전수 · 행 액션 · 일괄 작업 · 페이지네이션 · 상태 6종 · 권한.
- **생성**: 진입 경로 · 기본 정보(필드별 validation) · 입력 컴포넌트 본문 구성 · 종속 관계 · 저장/취소 · 권한 잠금 · 모달 · 확인 모달 · 상태 6종.
- **수정**: 진입 경로 · **잠금 정책(가장 누락 잦음)** · 변경 추적 · 저장 후 동작 · 상태 6종.
- **상세**: 헤더 액션 · 좌 기본 정보 전 필드 · 각 탭 전 섹션 · 테이블 전 컬럼 · 동선 · 상태 6종.
- **통합 폼**(`?mode=`): 모드별 편집 정책 매트릭스(섹션 × 모드) 필수.
- **상태 6종**: Loading / Empty(2종) / Partial / Error / Forbidden 명시.

## 🔴 오탐 금지 (관용 표기·의도된 강조는 위반 아님)

> 검수자가 정상 관용구·의도된 디자인 강조를 위반으로 잡으면 신뢰가 깨진다. 아래는 **위반으로 올리지 말 것**(메모리 *Figma Description 표기 관용·강조 피드백*):

- **`selected-default` = 기본 제공값 관용구** → D10/D5 위반 아님. 기본 제공/선택 기본값을 나타내는 표준 표기이므로 "허용 외 표기"로 잡지 않는다.
- **빨강 글자 = 의도된 최신 변경점 강조** → F9/D13 위반 아님. 빨강이 곧 "버전 마커/Changelog"는 아니다. **버전 토큰(`vX.X`)·NEW 배지·Changelog 프레임·일자 라인**이 실제로 있을 때만 F9. 단순 빨강 강조 텍스트는 통과.
- 위 둘은 **본문에 등장해도 PASS**. 다른 실제 위반(토큰 텍스트·NEW 배지 등)이 함께 있을 때만 그 항목으로 판정.
- **플랫 인코딩 구조 마커 = 정본 서식 → 위반 아님**: 소제목 `■`·글머리 `•`·연결자 `—`(em dash)·들여쓰기 공백(`0/2/5/8`). D9/D10/F7으로 잡지 않는다. 인라인 구분자 `·`(가운뎃점, clean·dirty)와 글머리 `•`는 **다른 글자**이니 혼동해 위반 처리 금지. 섹션 본문이 1개 TEXT 노드인 것은 **정상**(F6 = B 정본, 옛 "단일 노드 FAIL" 아님).

## 점검 누락 처리
- 읽기 hang/접근 불가로 못 본 항목 → 등급 `N/A(미점검)` + 사유. **PASS로 단정 금지.**
- 외부 SoT(용어집/상태값/공통정책) 미확인 → 해당 행 `WARN(SoT 미확인)`로 표시하고 리포트 메타에 기록.
