# 비판 카테고리별 시그널 사전

회의록·디자인에서 다음 패턴이 보이면 해당 카테고리에 적재한다.

## 1. 일정·스코프 리스크

- "고도화 시 반영", "4.x에서 제외", "다음 버전에"
- 출시일 임박 + 신규 요구 추가 패턴 (D-90 이내 식별자/지표 변경)
- "리팩토링 일정"이 GA 이후로 잡혀 있음
- 의사결정자 핫포테이토 (`@~확인 필요`가 회의록마다 반복)

## 2. 요구사항 정의의 구멍

- "고려 필요", "검토 필요", "정해야 함"이 N회 이상 회의록에 등장
- 같은 항목에 대해 회의록 A는 X, 회의록 B는 Y로 결정
- 디폴트 값이 명시되지 않은 토글/필터
- "공통 정책"이라 했다가 메뉴별 분기로 바뀐 항목

## 3. 화면 디자인 자체 문제

- raw JSON, raw 로그, mock 에러코드(`responseStatus: 4xx`)가 디자인에 노출
- 한 화면에 5개 이상 영역(네비/필터/표/차트/액션 등)
- 차트 타입이 BarChart/LineChart/AreaChart 등 3종 이상 혼재
- 캐주얼 톤(*"누가 ~함?"*) 툴팁
- 빈/로딩/에러 상태 화면 부재
- WA 인증/색맹/반응형 무언급

## 4. 용어 일관성 붕괴

- 같은 개념의 라벨이 회의록마다 다름 (예: 미터링/집계/이력)
- 회의에서 "변경 결정"했는데 디자인엔 이전 라벨이 그대로
- 영문/한글 용어 1:1 매핑 미확정

## 5. 거버넌스·문서화 리스크

- 회의록이 .txt / .rtf / UpNote / 하위 폴더에 산재
- 디자인 파일명에 버전·일자 없음 (`미터링.png`, `미터링-1.png`)
- "SoT가 어디인지" 회의록에 무언급
- 결정 SLA(예: 7일) 부재

## 6. 데이터·성능 리스크

- "분초단위 수집" + "월간 조회" 동시 명시
- CSV 다운로드 한계 무언급
- 권한별 데이터 마스킹 정책 부재
- "수집 누락 구간" 표시 무언급

## 7. 검증 누락

- 사용자 테스트(UT) 흔적 없음 — 모든 피드백이 내부 회의 안에서만 순환
- 수용 기준(AC) 부재 — "확대 기능 필요" 같은 모호한 결정만
- 기존 버전 → 신버전 마이그레이션 시나리오 무언급
- 롤백 시나리오 무언급
