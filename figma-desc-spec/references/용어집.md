# Product 기획 용어집 (controlled vocabulary)

Description 작성 시 **화면마다 동일 대상에 동일 용어**를 쓰기 위한 통제 어휘집. 약어 토큰(`U-10` 등) 대신 여기 표준 용어를 쓴다. 핵심 원칙 3 「동일 정의·동작 = 완전 동일 문구 유지」를 어휘 차원에서 강제.

> **소스(SoT 우선순위)**
> 1. **UI 컴포넌트 타입** = `planning-html/assets/공통템플릿/base.css` (클래스 + 한글 주석이 컴포넌트 분류의 권위) + 프로토타입 HTML.
> 2. **영역명·상태값·정책 용어** = 4.0 spec MD(`_참고자료/spec-format/`) + 상태값 xlsx.
> 3. **기존 기획 용어** = 3.0.6 full PDF(`매뉴얼/inputs/local_mirror/[3.0.6] Product_v8_20260421.pdf`) — ⏳ 심화 추출 다음 패스.
>
> 충돌 시: 컴포넌트 타입은 base.css, 정책 용어는 spec MD/Confluence 우선.

## 2. 영역/기능명 (섹션 title-bar 표준)

거터 옆 Title은 **영역 기능명** (핵심 원칙 3, 2026-06-30). 집계 보고서 확정 예:

`카테고리 Tab` · `필터 영역` · `조회 조건` · `집계 대상 선택` · `지표 선택` · `sticky 액션바` · `데이터 테이블` · `차트 카드`

## 4. 액션/정책 용어 (명사형 종결)

`적용` · `조회` · `저장` · `초기화` · `재조회` · `개별 해제` · `모두 제거` · `전체 선택/해제` · `일괄 처리`. 버튼은 `[적용]`·`[지표 저장]`처럼 대괄호.

## 5. 표기 관례 (핵심 원칙 3 + 2026-06-30)

- 버튼 `[버튼명]` · 라벨/UI명칭 `「라벨」` · **안내 문구/메시지 `"문구"`** · 브레드크럼/뎁스 `[상위 > 하위]`
- 항목명↔설명 연결 = **중간 대시 `—`** (온점 없이) · 한 줄 내 나열 구분 = `·` · 흐름 = `→`
- 목업 영역 마커 = `①②③` (거터 번호와 1:1·동기화)
- **`VM` 표기 금지 → `인스턴스`** (솔루션 내 통일)
- 이모지·장식 글리프 금지 (`💾 ▷ ⌬ ✓ ☐ …`)

## 6. 미결/참조 토큰 — 약어 배제 (2026-06-30)

미결·교차참조에 `U-10`·`T-11`·`F-16` 같은 **약어 토큰을 쓰지 않는다**(범례 없는 사적 약어). 대신 **짧은 한글 주제어**로: `모달 z-index 충돌 (미결) — …` · `대상 한도 (미결) — …` · `RAW 100행 cap 안내 (미결) — …`. (실제 적용: 집계 호스트 미결 블록 `[node-id]`, 2026-06-30.)

## 7. 3.0.6 계승 용어·관례 (대표 샘플, 2026-06-30)

3.0.6 full PDF(`local_mirror/[3.0.6] Product_v8_20260421.pdf`, 193p)에서 추출. ⚠️ **텍스트 레이어 깨짐(CID 폰트·ToUnicode 누락 → mojibake)** 으로 텍스트 마이닝 불가 → 페이지를 이미지 렌더 후 비전 OCR. 아래는 대표 스펙 페이지(p60 인스턴스 생성 폼, p80 연결 정보/스토리지) 샘플 — **전수 아님**.

**구조·번호 관례 (4.0과 동일 계승)**
- 섹션 번호 `N` + 하위 분할 `N-1`·`N-2` (p80: `6`·`6-1`·`6-2`). 4.0의 `1-1/1-2`와 동일.
- 본문 depth: `1.` 항목 → `a.` 세부 → `- ` 리프 디테일. → **3.0.6도 최하위에 대시 `- ` 사용**(사용자 "중간 대시" 규칙의 원류). 4.0 Figma는 네이티브 리스트로 렌더(`1./a./•`).
- 필수 필드 표기 `(필수)` (p60: `이름(필수)`·`프로젝트(필수)`).

**블록 라벨 변화 (3.0.6 → 4.0)**
- `정책` → **`동작 정책`** (p60의 `정책` 블록이 4.0에서 개명, 맨 뒤로).
- `Action Buttons` 섹션(취소/저장/확인) = 4.0의 공용 액션바·버튼 영역에 해당.
- 문체: 3.0.6은 경어체 혼재(`입력 가능합니다`) → **4.0 명사형 통일**(핵심 원칙 4).

**컴포넌트 표기 (3.0.6 용례)**
- `Radio Button`, `Table`, `Toggle (default-selected)/off`, `text field`, `Drop down` — 영문 표기 잔존(4.0 §1 표준 한글명으로 치환 대상).
- 연결 필드: `URL`·`AccessKey`·`Secret Key`·`Endpoint`·`Region`·`Bucket name`·`Path`.

**유효성 관례 (그대로 계승)**
- `영어 대소문자, 숫자, 한글, 언더바(_), 하이픈(-), 마침표(.) 만 입력 가능`
- `길이: N-M자` / `N~M자 사이 길이만 사용 가능`

**도메인 용어**
- 자원/대상: `인스턴스명`·`프로젝트(ID/명)`·`사용 유형`(일반(default)/GPU 전용)·`스냅샷`·`공급자`.
- 연결 대상: `Ceph`·`RabbitMQ`·`오브젝트 스토리지(S3)`·`Netapp`·`AWS`.

**표기/마커**
- 화면·메뉴 인용 `「화면명」으로 이동` (4.0의 `「」` 라벨 인용과 일치).
- 변경 마커: 빨강 글자 + 취소선 + `(vX.X.X 삭제/추가/수정)` + 날짜 (4.0 신규는 전역 history로 분리 → 인라인 미기재).
- `TBU`(To Be Updated) = 미확정 플레이스홀더 (4.0 미결 블록 `(미결)`에 해당).
