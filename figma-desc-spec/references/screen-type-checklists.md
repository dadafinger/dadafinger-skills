## 2. 분기·예외·Validation·동작·안내 4영역 상세 룰

전수 서술 의무에 더해, 다음 영역은 **항상 별도 섹션 또는 명시적 항목**으로 다룬다. 누락 시 개발 시점 clarification 비용이 가장 크다.

### 2-1. 화면 분기 관리 — 인라인 vs 별도 Screen ID

| 분기 규모 | 처리 방식 | 예시 |
|---|---|---|
| **작은 분기** (펼침/접힘·모달 오버레이·토글) | Description 본문에 **"분기 동작"** 항목 추가 | 행 펼침으로 조건 상세, [수정] 클릭 시 모달 |
| **중간 분기** (같은 URL·모드 차이) | **모드별 매트릭스** 표 | `?mode=create\|edit\|clone` |
| **큰 분기** (레이아웃 변경·별도 URL·권한별 차이) | **별도 Screen ID + 별도 페이지** | 관리자 vs 사용자, 데스크탑 vs 모바일 |

판단 기준: 같은 URL·쿼리에서 표시 차이만 → 인라인 / URL·쿼리·모드 변경 + 컴포넌트 추가·삭제 → 별도 ID 검토 / 권한별 노출 차이 큼 → 별도 ID 권고.

분기 케이스 다수 시 표기 예: `CONT-XX_01a 정상` · `CONT-XX_01b 권한 없음` · `CONT-XX_01c 빈 상태`.

### 2-2. 예외 상태 — 필수 6종

| 상태 | 필수 표기 항목 |
|---|---|
| **Loading** | 스켈레톤 패턴·진행률 임계(2초↑ progress)·타임아웃 임계 |
| **Empty (필터 결과 없음)** | 일러스트·안내 문구·"필터 완화" CTA·적용된 필터 재표시 |
| **Empty (데이터 없음, 신규 환경)** | 일러스트·"신규 생성" CTA·가이드 링크 |
| **Partial (일부 결측)** | 차트 빗금/셀 dim·상단 안내 배너 + 사유 |
| **Error (API/시스템 실패)** | 에러 코드·재시도 버튼·운영팀 링크·발생 시각 |
| **Forbidden (권한 없음)** | 안내 문구·권한 요청 채널 링크·대체 동선 |

> 6종 모두 디자인 시스템 표준 컴포넌트 사용. 화면별 직접 그리기 금지.

### 2-3. Validation (입력 필드별 5요소)

각 입력 필드는 다음 5가지를 모두 명시:

| 항목 | 표기 예 |
|---|---|
| 필수/선택 | `필수 *` · null 허용 여부 |
| 글자 수 제한 | 최소·최대 (1~50자) · 카운터 위치 (`12/50`) |
| 입력 형식 | 이메일·날짜(YYYY-MM-DD)·URL·IP/CIDR·정규식 |
| 에러 메시지 문구 | 정확한 한국어 ("필수 입력 항목입니다." / "최대 50자까지 입력 가능합니다.") |
| 에러 표시 위치·타이밍 | 인라인(필드 하단)/폼 상단 / blur·submit·typing 중 |

작성 양식:
```
- 알람 룰 이름 *필수
  - 최소 1자, 최대 50자
  - 형식: 자유 입력
  - 에러: "필수 입력 항목입니다." (submit 시) / "최대 50자까지 입력 가능합니다." (typing 중)
  - 표시: 필드 하단 인라인
```

### 2-4. 동적 컴포넌트 작동 방식

- **검색**: 대상 컬럼·LIKE/EQUAL/Fulltext·디바운스(300ms)·다중 검색 룰(컬럼 간 AND·같은 컬럼 OR)·키워드 추가 동작.
- **필터**: 그룹 내 OR / 그룹 간 AND·적용 칩·초기화 동작·**적용 즉시 vs [적용] 클릭**·URL 동기화.
- **정렬**: 정렬 가능 컬럼·기본 정렬축·오름/내림 토글·다중 정렬 여부.
- **페이지네이션 vs 무한 스크롤 vs 더보기** — 화면당 1개 방식 선택, 방식별 명시 항목 구분.
- **자동완성**: 최소 입력 글자·결과 N건 제한·로딩 인디케이터·키보드 탐색.
- **드래그·리사이즈**(있으면): 가능 여부·스냅 단위·제약.

### 2-5. 안내 팝업 & 토스트

- **이탈 방지 컨펌(Dirty Check)** — 생성·수정 화면 필수: 트리거(미저장 + 뒤로가기/탭 닫기/메뉴 이동/URL 변경)·문구("저장하지 않은 변경 사항이 있습니다. 정말 이동하시겠습니까?")·버튼 [이동]/[취소].
- **완료 토스트** — 등록·수정·삭제·일괄 후: 정확 문구("알람 룰이 생성되었습니다." / "{N}건의 알람 룰이 삭제되었습니다.")·위치·3~5초 자동 닫힘.
- **에러 토스트**: 문구("저장 중 오류가 발생했습니다. 다시 시도해 주세요.")·**수동 닫기**(자동 안 닫힘)·에러 코드 링크(선택).
- **확인 모달**(destructive): 처리 대상 명확화 + 결과 명시("{룰명} 알람 룰을 삭제하시겠습니까? 삭제해도 알람 센터의 발생 알람은 유지됩니다.")·예외 대상 제외 안내·destructive 색상.
- **안내 모달**(첫 사용·기능 변경): 자동 표시 정책(최초 1회/매번/N일)·"다시 보지 않기".

### 2-6. 4영역 검증
- [ ] 화면 분기가 인라인/별도 ID 중 적절히 처리됐는가
- [ ] 6가지 예외 상태가 모두 다뤄졌는가 (또는 N/A 사유)
- [ ] 모든 입력 필드의 Validation 5요소가 표기됐는가
- [ ] 모든 동적 컴포넌트의 작동 방식이 명시됐는가
- [ ] 이탈 방지·완료/에러 토스트·확인 모달 문구가 정확한 한국어로 적혀 있는가
