# benchmark-research

`리서치/` 폴더의 로컬 파일을 분석해 벤치마킹 MD를 만드는 스킬.

## 경로 상수

```
DESKTOP_BASE = ~/Desktop/
리서치 폴더  = DESKTOP_BASE/{버전}/{기능명}/리서치/
출력 폴더    = DESKTOP_BASE/{버전}/{기능명}/md/
```

## Mandatory Intake Gate

작업 시작 전 반드시 확정한다.

1. **기능명** — 예) 알람 설정
2. **버전** — 예) 4.0
3. `리서치/` 폴더 존재 및 파일 여부 확인
   - 폴더 없거나 비어 있으면 **중단**하고 파일 추가를 안내한다.

## 입력 파일 유형

| 확장자 | 처리 방식 |
|---|---|
| PNG / JPG / WEBP | 이미지 분석 — 화면 구성, 레이아웃, UX 패턴 위주 |
| MD | 텍스트 내용 통합 |
| PDF | 내용 발췌 |

파일명에서 서비스·사례명을 유추한다.

## 실행 규칙

- `리서치/` 폴더 내 파일만 입력으로 사용한다.
- 파일명 또는 폴더 구조에서 서비스명/사례명을 파악한다.
- 없는 서비스명은 "미상" 또는 파일명 그대로 표기한다.
- 기존 출력 파일이 있으면 덮어쓰지 않고 새 날짜로 저장한다.

## 출력

저장 경로: `DESKTOP_BASE/{버전}/{기능명}/md/벤치마킹_{기능명}_{YYYY-MM-DD}.md`

폴더 `md/`가 없으면 생성한다.

### 출력 포맷 템플릿

```markdown
# 벤치마킹 분석 — {기능명} ({버전}) {YYYY-MM-DD}

## 분석 대상
| # | 서비스·사례명 | 파일 |
|---|---|---|
| 1 | ... | ... |

## 주요 패턴 비교
| 항목 | {사례 A} | {사례 B} | ... |
|---|---|---|---|
| 레이아웃 | | | |
| 주요 기능 | | | |
| UX 흐름 | | | |
| 특이사항 | | | |

## 시사점 및 설계 방향
- ...

## 참조 파일 목록
- `리서치/{파일명}` — {사례명}
```
