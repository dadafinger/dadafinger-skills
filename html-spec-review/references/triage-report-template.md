# 트리아지 리포트 템플릿

목표: 기획자가 **읽고 accept/reject만** 하면 되도록. 막막한 전수 검수가 아니라 예/아니오 체크리스트로.

## 포맷

```
## {화면명} v{버전} · html ↔ MD 정합

판정: 일치 {M}/{전체} · 불일치 {N}건  (정본 = 라이브 html: {경로})

### 변경분/명세 일치 ✅ {M}/{전체}
| MD 주장 | 빌드(JSX) 실제 | |
|---|---|---|
| {변경분①/섹션 주장} | {디코드한 코드의 실제 문자열·동작} | ✅ |
| … | … | ✅ |

### ⚠️ 불일치 {N}건 — 결정 필요
| # | 항목 | MD | 빌드 | 정답 추정 | 결정 |
|---|---|---|---|---|---|
| 1 | {항목} | {MD 문구} | {빌드 문구/없음} | {어느 쪽이 맞을지 + 근거(과거 피드백/SoT)} | ☐ |
| 2 | … | … | … | … | ☐ |

### 🔧 footgun (있을 때만)
- 포크: 라이브 html ↔ {작업파일}이 {N}곳 다름 — {핵심 차이 요약}. 정본 후보: {…}
- 재패킹: {do_repack.py 등}이 {구버전 파일}을 가리킴 → 실행 시 라이브 퇴행 위험. **함부로 실행 금지.**
```

## 작성 원칙
- **자기선언 불신**: MD가 "반영 완료"라 해도 빌드 코드에서 직접 확인한 것만 ✅.
- **인용으로 근거**: 불일치는 빌드의 실제 문자열을 따온다(추측 금지).
- **정답 추정 + 근거**: 그냥 "다름"이 아니라 어느 쪽이 맞을지 + 왜(과거 피드백/용어집/유효성표 등 SoT)를 붙여 결정 부담을 낮춘다.
- **방향 분류**: 각 불일치는 ⓐ 빌드 정답→MD 수정 ⓑ MD 정답→html 패치 ⓒ 둘 다 틀림→양쪽 수정, 중 무엇인지 결정란이 가리키게.
- **사소/크로스파일 분리**: 화면 내부가 아닌 연계링크·머리말 같은 건 따로 묶어 "reject해도 무방"으로 표시.

## 예시 (알람 생성 v7)

```
## 알람 생성 v7 · html ↔ MD 정합
판정: 변경분 10/10 일치 · 불일치 3건  (정본 = 라이브 4. 알람 생성 v7.html)

### ⚠️ 불일치 3건
| # | 항목 | MD | 빌드 | 정답 추정 | 결정 |
|---|---|---|---|---|---|
| 1 | 개별 수신자 심각도 | "수신자별 심각도 오버라이드만" | 컨트롤 자체 제거 | 빌드(그룹만 제공이 의도) → MD 수정 | ☐ |
| 2 | 액션유예 안내 | "1건 집계+카운트 표시" | "통보하지 않습니다" | 둘 다 부정확 → "1건만 통보·빈도기준 카운트"로 양쪽 | ☐ |
| 3 | Stepper Step1 라벨 | "룰 정의" | "알람 룰 설정" | 라벨 텍스트 — 사용자 확정 | ☐ |
```
